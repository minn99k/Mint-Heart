# Capital Wars — 기능명세서

> 기획서 v6 기반. 각 기능의 상세 요구사항, 구현 순서, 의존관계, 역할분담 가이드를 정리한다.

---

## 1. 구현 순서 개요

### 왜 순서가 중요한가

기능 간 **의존관계**가 있다. SignalR 연결이 안 되면 방 생성을 테스트할 수 없고, 방이 없으면 게임을 시작할 수 없다. 아래 순서를 지키면 **매 단계마다 동작을 확인**할 수 있다.

### 4주 전체 흐름

```
Week 1: 연결 + 방 시스템
  → 확인: 브라우저 2개에서 방 입장, 게임 화면 진입

Week 2: 핵심 게임 로직 (서버)
  → 확인: 서버 로그로 유닛 생산/전송/상쇄/점령/승리 동작

Week 3: 프론트 연결 + 플레이 가능
  → 확인: 2명이 실제로 한 판을 끝까지 플레이

Week 4: 마무리 + 배포
  → 확인: Railway URL로 외부에서 게임 플레이
```

## 2. 기능 상세
### Phase 0: 인프라 세팅 (Week 1 전반)

> 게임 기능을 만들기 전에, 서버-클라이언트 간 **통신 파이프라인**을 먼저 뚫어야 한다. 이 단계가 완료되어야 이후 모든 기능의 테스트가 가능하다.

---

#### F-00-1. 서버 SignalR 등록

|항목|내용|
|---|---|
|설명|Server의 Program.cs에 SignalR 서비스 등록 + Hub 엔드포인트 매핑 + CORS 설정|
|구체적으로|`AddSignalR()`, `MapHub<GameHub>("/gamehub")`, 클라이언트 도메인 CORS 허용|
|확인 방법|서버 실행 시 에러 없이 `/gamehub` 엔드포인트가 열림|
|의존|없음 (가장 먼저)|
|구현 위치|Server — Program.cs|
|구현 순서|⭐ 0-1|

#### F-00-2. GameHub 껍데기 생성

|항목|내용|
|---|---|
|설명|Hub 클래스를 만들고, OnConnectedAsync / OnDisconnectedAsync에 로그만 추가|
|확인 방법|클라이언트가 연결하면 서버 콘솔에 "Player connected: {connectionId}" 출력|
|의존|F-00-1|
|구현 위치|Server — Hubs/GameHub.cs|
|구현 순서|⭐ 0-2|

#### F-00-3. 클라이언트 SignalR 연결

|항목|내용|
|---|---|
|설명|Client의 GameHubService에서 HubConnection을 생성하고 서버에 연결|
|구체적으로|`HubConnectionBuilder`로 서버 URL 지정, `StartAsync()`로 연결|
|확인 방법|브라우저 콘솔에 "Connected to server" 출력 + 서버 콘솔에 연결 로그|
|의존|F-00-2 (서버 Hub가 있어야 연결 대상이 있음)|
|구현 위치|Client — Services/GameHubService.cs|
|구현 순서|⭐ 0-3|

#### F-00-4. 양방향 통신 확인 (Ping-Pong)

|항목|내용|
|---|---|
|설명|클라이언트 → 서버 → 클라이언트 이벤트 왕복 테스트|
|구체적으로|Client가 `Ping` 전송 → GameHub가 받아서 `Pong` 응답 → Client가 수신 확인|
|확인 방법|브라우저에서 Pong을 받으면 양방향 통신 파이프 완성|
|의존|F-00-3|
|구현 위치|Server — GameHub / Client — GameHubService|
|구현 순서|⭐ 0-4|
|메모|테스트용 코드. 확인 후 제거하거나 주석 처리해도 됨|

#### F-00-5. 서비스 DI 등록 + Hub 연결

|항목|내용|
|---|---|
|설명|RoomService, GameService, MapLoader를 DI 컨테이너에 등록하고, GameHub에서 주입받아 사용하는 구조 세팅|
|구체적으로|Program.cs에 `AddSingleton<RoomService>()` 등 등록 → GameHub 생성자에서 주입|
|확인 방법|GameHub에서 서비스 메서드를 호출할 수 있음 (빈 메서드라도 호출 성공)|
|의존|F-00-4 (통신이 되어야 Hub를 통한 서비스 호출을 테스트 가능)|
|구현 위치|Server — Program.cs + GameHub.cs|
|구현 순서|⭐ 0-5|

## 4. 역할 분담 가이드

### 분담 원칙

- **Phase 0 + Week 1은 한 명(개발자A)이 주도**, 다른 한 명은 페어로 참여
- Week 2부터 분리: **같은 파일을 동시에 수정하지 않도록** 서비스 단위로 나눔
- GameLoopService(틱 처리)와 GameService(게임 로직)를 다른 사람이 맡으면 충돌이 줄어든다
