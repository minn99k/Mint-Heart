## 2. 게임 규칙

### 2-1. 한 줄 요약

> 내 노드에서 목표 노드로 파이프라인을 연결하면 유닛이 자동으로 흘러가서, 상대 유닛과 1:1로 상쇄하며 점령하는 실시간 전략 게임.

### 2-2. 게임 구조

- 맵은 **노드(도시)**들로 구성된다
- 게임 시작 시 각 플레이어에게 **시작 노드 1개**씩 배정, 나머지는 중립
- 플레이어는 **엣지(연결선)를 직접 생성**하여 유닛을 보낸다
- 상대의 **모든 노드를 점령**하면 승리

### 2-3. 노드

모든 노드는 항상 3가지 상태 중 하나:

| 상태 | 유닛 생산 | 설명 |
|------|-----------|------|
| **내 노드** | O (매초 +1) | 내가 점령한 노드 |
| **상대 노드** | O (매초 +1) | 상대가 점령한 노드 |
| **중립** | X | 아무도 점령하지 않은 노드. 초기 유닛만 보유 |

- 각 노드마다 **유닛 상한**이 다르다 (거점은 높고, 외곽은 낮음)
- 상한에 도달하면 유닛이 더 이상 생산되지 않음

### 2-4. 엣지(연결) 시스템

이 게임의 핵심 조작이다. 맵에 고정된 길이 아니라, **플레이어가 직접 만드는 파이프라인**.

#### 엣지 생성

- **내 노드** 클릭 → **아무 노드**(적/중립/아군) 클릭 → 연결 생성
- 연결 즉시 유닛 자동 전송 시작
- 한 노드에서 **최대 3개**까지 동시 연결 가능
- 같은 출발-도착 쌍은 1개만 존재 가능
- 자기 자신에게는 연결 불가

#### 엣지 해제

- **수동**: 연결된 출발 노드 → 도착 노드를 다시 클릭
- **자동**: 출발 노드가 상대에게 점령당하면, 해당 노드에서 나가는 모든 엣지가 자동 해제

#### 최대 3개 연결의 밸런스 의도

유닛 생산이 1유닛/초인데 연결 3개면 매초 3유닛이 빠져나간다. 즉 **연결을 많이 걸수록 노드가 빠르게 비워진다.** "공격에 몇 개, 지원에 몇 개 할당할지"를 판단하는 것이 전략의 핵심.

### 2-5. 유닛 전송

| 항목 | 내용 |
|------|------|
| 전송량 | 연결 1개당 1~2유닛/초 (추후 테스트 후 확정) |
| 전송 조건 | 출발 노드에 유닛이 1개 이상 있을 때 |
| 전송 차단 | 출발 노드 유닛이 0이면 자동 정지, 유닛 생기면 자동 재개 |
| 여러 연결 시 | 생성 순서대로 1유닛씩 돌아가며 분배 (라운드 로빈) |

#### 유닛 이동 속도

- 유닛은 **일정한 속도**로 이동한다 (예: 150px/초)
- 가까운 노드는 빨리 도착, 먼 노드는 오래 걸림
- 이동 속도는 맵 설정에서 조정 가능

### 2-6. 전투: 1:1 상쇄

이 게임에는 "전투 계산"이 없다. 모든 충돌은 **1:1 상쇄**로 처리된다.

#### 노드에서의 상쇄

내 유닛이 상대/중립 노드에 도착하면 → 도착 1개, 방어 1개가 동시에 사라짐

> **예시: 중립 노드 유닛 5개, 매초 내 유닛 1개 도착**
> 1초 후 → 5-1 = 4
> 2초 후 → 4-1 = 3
> ...
> 5초 후 → 1-1 = 0
> 6초 후 → 내 유닛 도착 → 점령 성공! 내 유닛 1개가 노드에 남음

#### 엣지 위에서의 상쇄

같은 경로에서 서로 반대 방향으로 이동하는 유닛이 만나면 → 1:1 상쇄

> **예시: 서울→대구 3유닛 이동 중, 대구→서울 5유닛 이동 중**
> → 3개씩 상쇄 → 대구→서울 2유닛만 생존하여 계속 이동

#### 아군 노드로의 전송

내 유닛이 내 노드에 도착 → 유닛 합산 (상한 적용)

#### 점령 규칙

| 상황 | 결과 |
|------|------|
| 도착 시 노드 유닛 > 0 (적/중립) | 1:1 상쇄, 소유권 변동 없음 |
| 도착 시 노드 유닛 = 0 (적/중립) | **점령 성공**, 소유권 전환 |
| 점령 성공 시 | 기존 소유자의 출발 엣지 전부 자동 해제 |

### 2-7. 승리 조건

| 조건 | 설명 |
|------|------|
| **점령 승리** | 상대의 소유 노드가 0개가 되면 즉시 승리 |
| **시간 승리** | 5분 경과 시 노드 수 → 유닛 수 순으로 비교 |
| **무승부** | 노드 수, 유닛 수 모두 동일 |

시간 제한(5분) 이유: 교착 상태 방지, 캐주얼 게임 특성상 한 판이 길면 이탈

### 2-8. 엣지케이스

#### 연결 끊김

| 상황 | 처리 |
|------|------|
| 게임 중 한쪽 끊김 | 30초 재접속 대기 (게임은 계속 진행) |
| 30초 내 재접속 | 이어서 진행 |
| 30초 초과 | 상대방 몰수승 |
| 대기방에서 나감 | 남은 사람 승리 |
| 양쪽 모두 끊김 | 둘 다 나가면 게임 무효 |

#### 예외 처리

| 상황 | 처리 |
|------|------|
| 상대 노드에서 엣지 생성 시도 | 거부 |
| 자기 자신으로 연결 시도 | 거부 |
| 노드당 3개 초과 연결 시도 | 거부 |
| 동일 엣지 중복 생성 시도 | 거부 |
| 유닛 0인 노드에서의 전송 | 자동 정지 (에러 아님) |
| 게임 종료 후 조작 | 거부 |

## 3. 맵 설계

### 3-1. 맵 구성 요소

| 요소 | 설명 |
|------|------|
| 노드 | 도시. 위치(x, y) + 초기 유닛 수 + 유닛 상한 + 소유자 |
| 유닛 속도 | 맵별로 설정하는 유닛 이동 속도 (px/초) |
| 시작 지점 | Player1, Player2의 시작 노드 지정 |

> 엣지는 맵 데이터에 없음 — 플레이어가 게임 중에 직접 생성

### 3-2. MVP 맵: 대한민국

**노드 구성 (12개):**

| 이름 | 초기 유닛 | 유닛 상한 | 역할 |
|------|-----------|-----------|------|
| 서울 | 10 | 30 | Player1 시작 |
| 인천 | 3 | 15 | 서울 인접 외곽 |
| 춘천 | 3 | 15 | 북부 외곽 |
| 강릉 | 2 | 10 | 동해안 외곽 |
| 청주 | 3 | 15 | 중부 연결 |
| 대전 | 4 | 20 | 중앙 거점 |
| 전주 | 3 | 15 | 서부 연결 |
| 광주 | 3 | 15 | 남서부 |
| 대구 | 4 | 20 | 중앙 거점 |
| 울산 | 3 | 15 | 동남부 |
| 부산 | 10 | 30 | Player2 시작 |
| 제주 | 2 | 10 | 남쪽 외곽 |

**노드 배치 (개략도):**

> 춘천 ──── 강릉
>
> 인천 ── 서울
> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;│
> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;청주
>
> 대전 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;대구 ── 울산
> &nbsp;&nbsp;│ &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;│
> 전주 &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;부산
> &nbsp;&nbsp;│
> 광주
> &nbsp;&nbsp;│
> 제주

**주요 경로별 이동 시간 (유닛 속도 150px/초 기준):**

| 경로 | 이동 시간 |
|------|-----------|
| 서울 → 인천 | ~0.8초 |
| 서울 → 청주 | ~0.7초 |
| 서울 → 대구 | ~1.9초 |
| 서울 → 부산 | ~2.8초 |
| 대전 → 대구 | ~1.5초 |
| 대구 → 부산 | ~0.9초 |

**밸런스 설계 의도:**
- 서울(P1)과 부산(P2)은 남북 끝에서 시작, 초기 유닛·상한 완전 대칭
- 대전/대구가 맵 중앙 거점 (초기 유닛과 상한이 높아 전략적 가치가 큼)
- 가까운 노드를 먼저 확보하는 것이 유리 (이동 시간이 짧으니까)
- 서울→부산 직접 연결은 2.8초로 기습 가능하지만 리스크 큼
- 제주는 외곽이라 가치 낮지만 추가 생산기지로 활용 가능

### 3-3. 향후 맵 추가 가이드라인

1. **대칭성**: 양쪽 시작 노드는 맵의 양 끝에 배치
2. **시작 조건 동일**: 초기 유닛, 상한 모두 대칭
3. **거리 균등**: 양쪽에서 중앙 거점까지 거리 유사
4. **노드 수**: 10~15개 권장
5. **상한 차등**: 거점(높음) > 일반(중간) > 외곽(낮음)
6. **맵 데이터 파일만 추가하면 됨**: 로직 변경 불필요

> │                                                │
> ├──────────────────────────────────────────────┤

**노드 색상:**
- 내 노드: 파랑(`#3B82F6`)
- 상대 노드: 빨강(`#EF4444`)

**노드 조작:**

| 동작 | 결과 |
|------|------|

| 선택 중 + 같은 노드 클릭 | 선택 해제 |
| 상대/중립 노드 클릭 (비선택) | 아무 동작 없음 |
| 최대 연결(3개) 상태에서 새 연결 | 에러 피드백 |

**엣지 시각화 (미확정, 테스트 후 결정):**
- 옵션 A: 엣지 선 없이, 이동 유닛(점)만 표시
- 옵션 B: 얇은 점선으로 엣지 표시 + 이동 유닛
- 옵션 C: 내 연결만 표시, 상대 연결은 이동 유닛으로만

## 5. 워크플로우 (유저 시나리오)

> **이 장의 목적:** 유저 입장에서 "나는 뭘 하고, 그러면 뭐가 일어나는지"를 시간 순서로 정리.
> 이 시나리오들이 곧 "우리가 만들어야 할 것의 전부"이다.

### 5-1. 방 만들기 → 상대 대기 → 게임 시작

1. 로비 화면에 접속한다
2. 닉네임을 입력한다
3. "방 만들기" 버튼을 누른다
4. 대기 화면으로 이동한다
5. 화면에 방 코드(6자리 숫자)가 크게 표시된다
6. 이 코드를 친구에게 알려준다 (카톡 등)
7. "상대를 기다리는 중..." 텍스트가 보인다
8. 친구가 입장하면 상대 닉네임이 표시된다
9. "곧 시작합니다!" + 3초 카운트다운이 시작된다
10. 카운트다운 완료 → 게임 화면으로 전환된다

### 5-2. 방 참가 → 게임 시작

1. 로비 화면에 접속한다
2. 닉네임을 입력한다
3. 친구에게 받은 방 코드를 입력한다
4. "참가" 버튼을 누른다
5. 대기 화면으로 이동한다
6. 상대(방장) 닉네임이 이미 표시되어 있다
7. 3초 카운트다운이 시작된다
8. 카운트다운 완료 → 게임 화면으로 전환된다

### 5-3. 게임 시작 직후 — 맵 상태 확인

1. 게임 화면이 열린다
2. 맵에 12개 노드(원)가 보인다
3. 내 시작 노드(서울)가 파랑, 상대(부산)가 빨강, 나머지 10개가 회색
4. 각 노드 안에 유닛 수가 표시된다
5. 상단 HUD에 내 유닛 총합, 상대 유닛 총합, 경과 시간이 보인다
6. 매초 내 노드(서울)의 유닛이 +1씩 올라간다 (자동 생산)
7. 게임은 이미 시작되었다 — 빨리 행동해야 한다

### 5-4. 엣지 연결 → 유닛 전송 → 노드 점령

1. 서울(파랑, 10유닛)을 클릭한다 → 서울이 하이라이트된다
2. 인천(회색, 3유닛)을 클릭한다 → 연결 생성!
3. 서울에서 매초 유닛 1개가 빠져나간다 (10→9→8...)
4. 작은 파란 점이 서울에서 인천 방향으로 이동하는 게 보인다
5. 약 0.8초 후 첫 유닛이 인천에 도착한다
6. 인천 유닛이 줄어든다 (3→2→1→0)
7. 인천 유닛이 0이 되고, 다음 유닛이 도착하면 → 점령 성공!
8. 인천이 파랑으로 변한다
9. 인천에서도 매초 유닛이 자동 생산되기 시작한다
10. 연결은 계속 유지 중 → 서울에서 인천으로 유닛이 계속 흘러간다 (인천 상한에 도달하면 합산되지 않고 넘침 방지)

### 5-5. 여러 노드에 동시 연결

1. 서울을 클릭한다
2. 인천을 클릭한다 → 연결 1 생성
3. 서울을 다시 클릭한다
4. 청주를 클릭한다 → 연결 2 생성
5. 서울을 다시 클릭한다
6. 대구(빨강)를 클릭한다 → 연결 3 생성 (적 노드로도 연결 가능!)
7. 서울에서 매초 3유닛이 빠져나간다 (각 연결에 1유닛씩)
8. 서울은 매초 +1 생산, -3 전송 = -2/초 → 빠르게 줄어든다
9. 서울을 다시 클릭 → 춘천 클릭 → "최대 연결(3개) 초과!" 에러 피드백

### 5-6. 엣지 해제

1. 서울→인천 연결이 활성 중이다
2. 서울을 클릭한다 → 하이라이트
3. 인천을 클릭한다 → 이미 연결되어 있으므로 연결 해제!
4. 서울에서 인천으로 가던 유닛 전송이 중단된다
5. 이미 이동 중이던 유닛은 목적지까지 계속 이동한다
6. 서울의 연결이 2개로 줄어든다 → 새 연결 1개 가능

### 5-7. 엣지 위 상쇄

1. 나(Player1): 서울→대구 연결 활성 → 유닛이 계속 이동 중
2. 상대(Player2): 대구→서울 연결 활성 → 유닛이 반대로 이동 중
3. 같은 경로에서 서로 반대 방향으로 이동하는 유닛이 만난다
4. 1:1로 상쇄 → 둘 다 사라짐
5. 한쪽이 더 많이 보내고 있으면, 남은 유닛만 목적지에 도착
6. 화면에서: 두 색의 점이 만나면 "퍽" 하고 사라지는 느낌

### 5-8. 노드 점령당함 → 엣지 자동 해제

1. 내가 대전을 점령하고, 대전→광주 연결을 걸어둔 상태
2. 상대가 대전으로 유닛을 계속 보낸다
3. 대전의 내 유닛이 줄어든다 (10→5→3→1→0)
4. 대전이 상대에게 점령당한다 (빨강으로 변함)
5. 내가 걸어둔 "대전→광주" 연결이 자동으로 해제된다
6. 대전은 이제 상대 노드이므로, 내가 대전에서 연결을 걸 수 없다

### 5-9. 승리

**점령 승리**
1. 상대의 마지막 노드를 점령한다
2. 상대의 소유 노드가 0개가 된다
3. 즉시 게임 종료 → "승리!" 표시
4. 결과 화면으로 전환

**시간 승리**
1. 5분이 경과한다
2. 보유 노드 수 비교 → 내가 많으면 승리
3. 노드 수 동일 시 → 총 유닛 수 비교
4. 둘 다 동일 → 무승부
5. 결과 화면으로 전환

### 5-10. 게임 끝 → 다시하기 / 로비

1. 결과 화면에 최종 맵 상태, 점수, 시간이 표시된다
2. "다시 하기" 클릭 → 같은 맵으로 새 방 생성 → 대기 화면
3. "로비로" 클릭 → 로비 화면으로 이동

### 5-11. 연결 끊김

**상대가 끊김**
1. 게임 중 상대의 연결이 끊긴다
2. 화면에 "상대방의 연결이 끊겼습니다. 30초 대기 중..." 표시
3. 게임은 계속 진행된다 (유닛 생산, 전송 모두 계속)
4-A. 30초 내 재접속 → "상대방이 돌아왔습니다" → 게임 계속
4-B. 30초 초과 → 내가 몰수승 → 결과 화면

**내가 끊김**
1. 네트워크가 끊긴다
2. 재접속하면 게임이 진행 중이던 상태 그대로 이어진다
3. 끊긴 동안 내 엣지와 유닛 생산은 계속 돌아가고 있었다

### 5-12. 방 참가 실패

1. 로비에서 방 코드 "999999"를 입력하고 참가를 누른다
2. "존재하지 않는 방입니다" 에러 메시지가 표시된다
3. 화면은 로비에 그대로 남아있다
4. 올바른 코드를 다시 입력해서 참가할 수 있다
