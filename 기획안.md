# Capital Wars — 기획서

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 한 줄 설명 | 내 노드에서 목표 노드로 유닛을 보내면 자동으로 흘러가서, 상대와 1:1 상쇄하며 점령하는 실시간 1v1 웹 전략 게임 |
| 배경 | Sperm Wars(iOS) 스타일 세포 전략 게임을 웹에서 친구와 즉시 플레이하고 싶음 |
| 목표 | 실제 서버에 배포, 2명이 브라우저에서 실시간 대전 가능 |
| 대상 사용자 | 링크 하나로 바로 플레이하고 싶은 캐주얼 유저 |
| 범위 | ✅ 1v1 대전, 대한민국 맵 1개 / ❌ 랭킹, 로그인, 관전, 모바일 최적화, 맵 에디터 |
| 제약 | 4주, 2명(기능 단위 풀스택), 예산 0원, Blazor WASM + ASP.NET Core + SignalR, Railway 배포 |

## 2. 핵심 규칙

### 기본 구조

- 맵은 **12개 노드(도시)**로 구성
- 각 플레이어에게 시작 노드 1개, 나머지 10개는 중립
- 로그인 없음, 닉네임(최대 12자)만 입력
- **플레이 타임 5분**

### 노드

- 점령한 노드는 **매초 유닛 +1** 자동 생산 (노드별 상한 있음, 중립은 생산 없음)
- 노드 상한 차등: 시작 노드(30) > 거점(20) > 일반(15) > 외곽(10)
- 노드가 유닛을 생성하고 보유량을 관리

### 전송 (핵심 조작)

유닛은 출발지와 도착지를 알고 스스로 이동한다. 별도의 "엣지" 객체 없이, **"이 노드에서 저 노드로 유닛을 계속 보내라"는 전송 명령**으로 동작.

- **전송 시작**: 내 노드 클릭 → 아무 노드 클릭 → 즉시 유닛 자동 전송 시작
- **전송 중단**: 같은 노드 쌍을 다시 클릭
- **자동 중단**: 출발 노드를 적에게 빼앗기면 해당 노드의 모든 전송 중단
- **제한**: 노드당 최대 3개 전송, 자기 자신 불가, 동일 쌍 중복 불가

### 유닛

- 연결당 1~2유닛/초 자동 전송 (추후 테스트 후 확정)
- 출발 노드 유닛 0이면 전송 자동 정지, 생기면 자동 재개
- 여러 전송 시 생성 순서대로 라운드 로빈 분배
- 유닛 이동 속도 고정 (예: 150px/초), 먼 노드는 자연스럽게 오래 걸림

### 전투 (1:1 상쇄)

- **노드 도착**: 적/중립 유닛과 1:1 상쇄, 유닛 0인 노드에 도착하면 점령
- **이동 중 충돌**: 서로 다른 경로에서 상대 유닛을 만나면 1:1 상쇄, 같은 편 유닛끼리는 기존 경로 유지
- **아군 도착**: 합산 (상한 적용)
- 점령 성공 시 기존 소유자의 전송 전부 자동 중단

### 승리/종료

- **점령 승리**: 상대 노드 0개 → 즉시 승리
- **시간 승리**: 5분 경과 → 노드 수 비교 → 동률 시 유닛 수 비교
- **연결 끊김**: 30초 재접속 대기(게임 계속 진행), 초과 시 상대 몰수승

## 3. 구성 요소

### 노드

맵을 이루는 거점. 유닛을 생산하고 보유한다.

| 속성 | 설명 |
|------|------|
| 이름 | 도시 이름 (서울, 부산 등) |
| 보유 유닛 수 | 현재 노드에 있는 유닛 수 |
| 유닛 상한 | 노드별 최대 보유 가능 수 |
| 초기 유닛 | 게임 시작 시 유닛 수 (중립 노드의 점령 문턱) |
| 소유자 | Player1 / Player2 / 중립 |
| 위치 | 맵 상 좌표 (x, y) |

**대한민국 맵 (12개):**

| 이름 | 초기 유닛 | 상한 | 역할 |
|------|-----------|------|------|
| 서울 | 10 | 30 | P1 시작 |
| 부산 | 10 | 30 | P2 시작 |
| 대전, 대구 | 4 | 20 | 중앙 거점 |
| 인천, 춘천, 청주, 전주, 광주, 울산 | 3 | 15 | 일반 |
| 강릉, 제주 | 2 | 10 | 외곽 |

### 유닛

노드가 생산하는 전투 단위. 독립적인 개체로 이동한다.

| 속성 | 설명 |
|------|------|
| 소유자 | 어느 플레이어의 유닛인지 |
| 상태 | 노드에 대기 중 / 이동 중 / 소멸 |
| 출발 노드 | 어디서 출발했는지 |
| 도착 노드 | 어디로 향하는지 |
| 이동 속도 | 고정값 (예: 150px/초) |
| 현재 위치 | 이동 중일 때의 좌표 |

## 4. 화면/흐름

> [로비] → [대기] → [게임] → [결과] → [로비]

### 로비

- **목적**: 방을 만들거나 참가
- **요소**: 닉네임 입력, 방 만들기 버튼, 방 코드(숫자 6자리) 입력, 참가 버튼
- **행동**: 닉네임 입력 후 방 만들기 or 코드 입력 후 참가
- **전환**: 성공 → 대기 화면 / 실패(방 없음, 꽉참, 닉네임 미입력) → 에러 표시, 화면 유지

### 대기

- **목적**: 상대방 접속 대기
- **요소**: 방 코드(크게), 맵 이름, 양쪽 닉네임, 상태 텍스트
- **전환**: 2명 접속 → 3초 카운트다운 → 게임 / 한쪽 나감 → 남은 사람 승리

### 게임

- **목적**: 메인 플레이
- **레이아웃**:

> ┌──────────────────────────────────────────┐
> │ 🔵 P1: 45유닛/4노드  🔴 P2: 38/3  ⏱2:34│
> ├──────────────────────────────────────────┤
> │           [SVG 맵 영역]                   │
> │  노드(원) + 유닛 수 + 이동 유닛(점)      │
> └──────────────────────────────────────────┘

- **색상**: 내 노드 파랑 / 상대 빨강 / 중립 회색
- **조작**: 내 노드 클릭(선택) → 다른 노드 클릭(전송 시작 or 중단) → 같은 노드 재클릭(선택 해제)

### 결과

- **목적**: 승패 확인 + 재시작
- **요소**: 승리/패배/무승부, 최종 맵 스냅샷, 시간, 점수(노드 수·유닛 수)
- **행동**: 다시 하기(→ 대기) or 로비로(→ 로비)

## 5. 워크플로우

### 시나리오 1: 방 만들기 → 게임 시작

> 1. 닉네임 입력 → "방 만들기" 클릭
> 2. 대기 화면에 방 코드(예: 482917) 표시
> 3. 친구에게 코드 공유 → 친구가 코드 입력 후 참가
> 4. 양쪽 화면에 상대 닉네임 표시 → 3초 카운트다운
> 5. 게임 시작 — 맵에 12개 노드, 내 노드(파랑), 상대(빨강), 중립(회색)

### 시나리오 2: 핵심 플레이 — 전송 → 점령

> 1. 서울(내 노드, 10유닛) 클릭 → 하이라이트
> 2. 인천(중립, 3유닛) 클릭 → 전송 시작, 유닛이 자동으로 흘러감
> 3. 서울에서 매초 1유닛 빠져나감 → 파란 점이 인천으로 이동
> 4. 첫 유닛 도착 → 인천 3→2→1→0 상쇄
> 5. 다음 유닛 도착 → 점령! 인천이 파랑으로 변하고 유닛 생산 시작
> 6. 서울 클릭 → 청주 클릭 → 두 번째 전송 (동시 2곳 공격)
> 7. 서울 매초 +1 생산, -2 전송 = -1/초 → 유닛이 줄어든다

### 시나리오 3: 이동 중 상쇄 + 노드 빼앗김

> **[이동 중 상쇄]**
> 1. 나: 서울→대구 전송 중, 상대: 대구→서울 전송 중
> 2. 이동 중인 유닛끼리 만남 → 1:1 상쇄, 양쪽 소멸
> 3. 더 많이 보내는 쪽 유닛만 살아남아 도착
>
> **[노드 빼앗김 → 자동 중단]**
> 4. 상대 유닛이 내 대전에 계속 도착 → 대전 유닛 감소
> 5. 대전 유닛 0 → 상대 유닛 도착 → 점령당함 (빨강으로 변함)
> 6. 내가 걸어둔 대전→광주 전송이 자동 중단됨

### 시나리오 4: 게임 종료

> **[점령 승리]**
> 1. 상대의 마지막 노드를 점령 → 상대 노드 0개 → 즉시 승리
> 2. 결과 화면: "승리!", 최종 맵, 점수
>
> **[시간 승리]**
> 1. 5분 경과 → 노드 수 비교 → 동률 시 유닛 수 비교
> 2. 결과 화면: 승/패/무승부
>
> **[연결 끊김]**
> 1. 상대 끊김 → "30초 대기 중..." 표시 (게임은 계속)
> 2-A. 재접속 → 이어서 진행
> 2-B. 30초 초과 → 내가 몰수승
