# Capital Wars — 기획서

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 한 줄 설명 | 내 노드에서 목표 노드로 파이프라인을 연결하면 유닛이 자동으로 흘러가서, 상대와 1:1 상쇄하며 점령하는 실시간 1v1 웹 전략 게임 |
| 배경 | Sperm Wars(iOS) 스타일 세포 전략 게임을 웹에서 친구와 즉시 플레이하고 싶음 |
| 목표 | 실제 서버에 배포, 2명이 브라우저에서 실시간 대전 가능 |
| 대상 사용자 | 링크 하나로 바로 플레이하고 싶은 캐주얼 유저 |
| 범위 | ✅ 1v1 대전, 대한민국 맵 1개 / ❌ 랭킹, 로그인, 관전, 모바일 최적화, 맵 에디터 |
| 제약 | 4주, 2명(기능 단위 풀스택), 예산 0원, Blazor WASM + ASP.NET Core + SignalR, Railway 배포 |

## 2. 핵심 규칙

### 기본 구조

- 맵은 **12개 노드(도시)**로 구성
- 게임 시작 시 각 플레이어에게 시작 노드 1개, 나머지 10개는 중립
- 로그인 없음, 닉네임(최대 12자)만 입력

### 노드

- 점령한 노드는 **매초 유닛 +1** 자동 생산 (노드별 상한 있음, 중립은 생산 없음)
- 노드 상한 차등: 시작 노드(30) > 거점(20) > 일반(15) > 외곽(10)

### 엣지 (핵심 조작)

- **생성**: 내 노드 클릭 → 아무 노드 클릭 → 연결, 즉시 자동 전송 시작
- **해제**: 같은 노드 쌍을 다시 클릭
- **자동 해제**: 출발 노드를 적에게 빼앗기면 해당 노드의 모든 엣지 해제
- **제한**: 노드당 최대 3개, 자기 자신 불가, 동일 쌍 중복 불가

### 유닛 전송

- 연결당 1~2유닛/초 자동 전송 (추후 테스트 후 확정)
- 출발 노드 유닛 0이면 자동 정지, 생기면 자동 재개
- 여러 연결 시 생성 순서대로 라운드 로빈 분배
- 유닛 이동 속도 고정 (예: 150px/초), 먼 노드는 자연스럽게 오래 걸림

### 전투 (1:1 상쇄)

- **노드 도착**: 적/중립 유닛과 1:1 상쇄, 유닛 0인 노드에 도착하면 점령
- **엣지 위**: 반대 방향 유닛끼리 만나면 1:1 상쇄
- **아군 도착**: 합산 (상한 적용)
- 점령 성공 시 기존 소유자의 출발 엣지 전부 자동 해제

### 승리/종료

- **점령 승리**: 상대 노드 0개 → 즉시 승리
- **시간 승리**: 5분 경과 → 노드 수 비교 → 동률 시 유닛 수 비교
- **연결 끊김**: 30초 재접속 대기(게임 계속 진행), 초과 시 상대 몰수승

## 3. 구성 요소

### 노드 (대한민국 맵, 12개)

| 이름 | 초기 유닛 | 상한 | 역할 |
|------|-----------|------|------|
| 서울 | 10 | 30 | P1 시작 |
| 부산 | 10 | 30 | P2 시작 |
| 대전, 대구 | 4 | 20 | 중앙 거점 |
| 인천, 춘천, 청주, 전주, 광주, 울산 | 3 | 15 | 일반 |
| 강릉, 제주 | 2 | 10 | 외곽 |

### 노드 배치

```
        춘천 ──── 강릉
인천 ── 서울
          │
        청주
대전            대구 ── 울산
  │               │
전주            부산
  │
광주
  │
제주
```

### 주요 거리 (유닛 속도 150px/초 기준)

| 경로 | 시간 | 경로 | 시간 |
|------|------|------|------|
| 서울→인천 | 0.8초 | 대전→대구 | 1.5초 |
| 서울→청주 | 0.7초 | 대구→부산 | 0.9초 |
| 서울→대구 | 1.9초 | 서울→부산 | 2.8초 |

### 밸런스 설계

- 서울-부산 완전 대칭 (초기 유닛, 상한 동일)
- 중앙 거점(대전/대구)이 높은 상한 → 중앙 쟁탈 유도
- 가까운 노드 선점이 유리 (이동 시간이 짧으니까)
- 서울→부산 직접 연결(2.8초)은 기습 가능하지만 리스크 큼

### 엣지

- 맵 데이터에 없음. 게임 중 플레이어가 직접 생성
- 모든 노드 쌍 사이에 연결 가능 (거리에 따라 이동 시간만 달라짐)

## 4. 화면/흐름

```
[로비] → [대기] → [게임] → [결과] → [로비]
```

### 로비

- **목적**: 방을 만들거나 참가
- **요소**: 닉네임 입력, 방 만들기 버튼, 방 코드(숫자 6자리) 입력, 참가 버튼
- **행동**: 닉네임 입력 후 방 만들기 or 코드 입력 후 참가
- **전환**: 성공 → 대기 화면 / 실패(방 없음, 꽉참, 닉네임 미입력) → 에러 표시, 화면 유지

### 대기

- **목적**: 상대방 접속 대기
- **요소**: 방 코드(크게), 맵 이름, 양쪽 닉네임, 상태 텍스트
- **전환**: 2명 접속 → 3초 카운트다운 → 게임 / 한쪽 나감 → 남은 사람 승리

### 게임

- **목적**: 메인 플레이
- **레이아웃**:

```
┌──────────────────────────────────────────┐
│ 🔵 P1: 45유닛/4노드  🔴 P2: 38/3  ⏱2:34│
├──────────────────────────────────────────┤
│           [SVG 맵 영역]                   │
│  노드(원) + 유닛 수 + 이동 유닛(점)      │
├──────────────────────────────────────────┤
│ 서울(12) 선택 중 → 목표 노드를 클릭      │
└──────────────────────────────────────────┘
```

- **색상**: 내 노드 파랑 / 상대 빨강 / 중립 회색
- **조작**: 내 노드 클릭(선택) → 다른 노드 클릭(연결 생성 or 해제) → 같은 노드 재클릭(선택 해제)
- **엣지 시각화**: 미확정 (선 없이 유닛만 / 점선+유닛 / 내 연결만 표시 — 테스트 후 결정)

### 결과

- **목적**: 승패 확인 + 재시작
- **요소**: 승리/패배/무승부, 최종 맵 스냅샷, 시간, 점수(노드 수·유닛 수)
- **행동**: 다시 하기(→ 대기) or 로비로(→ 로비)

## 5. 워크플로우

### 시나리오 1: 방 만들기 → 게임 시작

```
1. 닉네임 입력 → "방 만들기" 클릭
2. 대기 화면에 방 코드(예: 482917) 표시
3. 친구에게 코드 공유 → 친구가 코드 입력 후 참가
4. 양쪽 화면에 상대 닉네임 표시 → 3초 카운트다운
5. 게임 시작 — 맵에 12개 노드, 내 노드(파랑), 상대(빨강), 중립(회색)
```

### 시나리오 2: 핵심 플레이 — 연결 → 전송 → 점령

```
1. 서울(내 노드, 10유닛) 클릭 → 하이라이트
2. 인천(중립, 3유닛) 클릭 → 연결 생성, 유닛 자동 전송 시작
3. 서울에서 매초 1유닛 빠져나감 → 파란 점이 인천으로 이동
4. ~0.8초 후 첫 도착 → 인천 3→2→1→0 상쇄
5. 다음 유닛 도착 → 점령! 인천이 파랑으로 변하고 유닛 생산 시작
6. 서울 클릭 → 청주 클릭 → 두 번째 연결 (동시 2곳 공격)
7. 서울 매초 +1 생산, -2 전송 = -1/초 → 유닛이 줄어든다
```

### 시나리오 3: 엣지 위 상쇄 + 노드 빼앗김

```
[엣지 위 상쇄]
1. 나: 서울→대구 연결 중, 상대: 대구→서울 연결 중
2. 같은 경로에서 반대 유닛이 만남 → 1:1 상쇄, 양쪽 소멸
3. 더 많이 보내는 쪽 유닛만 살아남아 도착

[노드 빼앗김 → 자동 해제]
4. 상대 유닛이 내 대전에 계속 도착 → 대전 유닛 감소
5. 대전 유닛 0 → 상대 유닛 도착 → 점령당함 (빨강으로 변함)
6. 내가 걸어둔 대전→광주 연결이 자동 해제됨
```

### 시나리오 4: 게임 종료

```
[점령 승리]
1. 상대의 마지막 노드를 점령 → 상대 노드 0개 → 즉시 승리
2. 결과 화면: "승리!", 최종 맵, 점수

[시간 승리]
1. 5분 경과 → 노드 수 비교 → 동률 시 유닛 수 비교
2. 결과 화면: 승/패/무승부

[연결 끊김]
1. 상대 끊김 → "30초 대기 중..." 표시 (게임은 계속)
2-A. 재접속 → 이어서 진행
2-B. 30초 초과 → 내가 몰수승
```

## 6. 기능 명세

### 로비/방

| ID | 기능 | 우선순위 |
|----|------|----------|
| F-01 | 닉네임 입력 (최대 12자, 필수) | 필수 |
| F-02 | 방 생성 (숫자 6자리 코드) | 필수 |
| F-03 | 방 참가 (코드 입력, 실패 시 에러) | 필수 |
| F-04 | 대기 화면 (코드 표시, 상대 대기, 카운트다운) | 필수 |

| ID | 기능 | 우선순위 |
|----|------|----------|
| F-05 | 맵 렌더링 (SVG 노드 + 유닛 수 + 색상) | 필수 |
| F-06 | 노드 선택 + 하이라이트 | 필수 |
| F-07 | 엣지 생성 (선택 + 다른 노드 클릭) | 필수 |

| ID | 기능 | 우선순위 |
